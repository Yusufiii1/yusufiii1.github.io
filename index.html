Projectplan — 2026

Strawberry π

Een volledig geautomatiseerde mini-serre met 2 geïsoleerde kamers, aangestuurd door een Raspberry Pi 4 — inclusief remote monitoring website.

Platform Raspberry Pi 4 (4GB)
Experiment 24/7 licht vs. 12-13u licht
Monitoring Website + Livestream + Timelapse
Kamer A

24/7 Continu Licht

Aardbeien krijgen de hele dag en nacht licht. Kweeklamp staat altijd aan via relay module.

Kamer B

Normale Dagcyclus

Aardbeien krijgen 12–13 uur licht per dag. Kweeklamp wordt automatisch aan/uit gezet door de Pi.

Stappenplan
1

Pi Voorbereiden

Systeem updaten en libraries installeren

Hardware
01

Systeem updaten

Run sudo apt update && sudo apt upgrade op de Pi.

02

Python libraries installeren

Installeer: flask, opencv-python, adafruit-dht, adafruit-ads1x15, RPi.GPIO, smbus2

03

Cameras testen

Beide USB cameras aansluiten en controleren of ze herkend worden via ls /dev/video*

04

I2C inschakelen

Via sudo raspi-config → Interface Options → I2C inschakelen (voor ADS1115 en LCD).

2

24/7 Control Script

Automatisatie + data wegschrijven naar JSON

Software
01

Sensor uitlezing schrijven

DHT22 (temp + humidity) en ADS1115 (soil moisture) uitlezen voor beide kamers.

02

Fan automatisatie

PWM sturing via L298N op basis van temp/humidity drempelwaarden. 3 niveaus: 0%, 60%, 100%.

03

Waterpomp automatisatie

Elke 4 uur soil check. Als voltage >1.46V → pomp aan voor (1.71 − V) × 30 seconden.

04

Licht automatisatie

Kamer A: relay altijd aan. Kamer B: relay aan/uit op basis van tijdschema (12–13u licht).

05

Data wegschrijven naar JSON

Na elke cyclus schrijft het script alle data naar data.json. Flask leest dit bestand.

06

Log bijhouden

Elke actie wordt gelogd met timestamp naar log.json.

07

LCD display updaten

Grove 16×2 LCD toont live temp + humidity van beide kamers.

3

Flask API Server

Backend die data deelt en commando's ontvangt

Software
01

Flask installeren en basis opzetten

Aparte api.py file die naast het control script draait op poort 5000.

02

GET endpoint: sensordata

/api/data → geeft alle sensordata terug als JSON aan de website.

03

GET endpoint: log

/api/log → geeft de volledige log terug (fan events, pomp events, etc.).

04

POST endpoint: emergency commands

/api/command → ontvangt commando's. Beveiligd met wachtwoord.

05

Command handler schrijven

Flask schrijft commando naar command.json. Control script voert het uit zonder automatisatie te onderbreken.

4

Camera Streams & Timelapse

MJPEG livestream + automatische timelapse

Software
01

MJPEG stream server schrijven

Via OpenCV en Flask: camera 1 op /stream/cam1 en camera 2 op /stream/cam2. ~5-10fps op lage resolutie.

02

Stream lokaal testen

Via browser op zelfde netwerk controleren of beide streams werken.

03

Timelapse script schrijven

Aparte timelapse.py die elke keer een foto trekt van beide cameras en opslaat als YYYY-MM-DD_HH-00.jpg.

04

Timelapse automatiseren via cron

0 * * * * python3 /home/pi/timelapse.py → elke uur een foto. 24fps = 1 seconde per dag.

05

Timelapse video exporteren (achteraf)

ffmpeg -framerate 24 -pattern_type glob -i 'cam1/*.jpg' timelapse_cam1.mp4

5

Cloudflare Tunnel

Pi bereikbaar maken van buiten het netwerk

Netwerk
01

Cloudflare account aanmaken

Gratis account op cloudflare.com. Geen creditcard nodig.

02

cloudflared installeren op Pi

Download en installeer de cloudflared ARM binary op de Pi.

03

Tunnel aanmaken en configureren

Routes instellen: /api/* → Flask (poort 5000), /stream/* → Camera server.

04

Tunnel als service instellen

Cloudflare Tunnel automatisch laten starten bij Pi reboot via systemd service.

05

Publieke URL testen

Controleren of API en camera streams bereikbaar zijn van buiten het thuisnetwerk.

6

Website (GitHub Pages)

Frontend dashboard met alle info

Website
01

GitHub repository aanmaken

Nieuwe repo aanmaken en GitHub Pages inschakelen op username.github.io/strawberry-pi.

02

Dashboard pagina

Live sensordata van beide kamers: temp, humidity, soil moisture, fan status, licht status. Auto-refresh elke 10 seconden.

03

Camera pagina

2 MJPEG streams naast elkaar via <img src="https://tunnel-url/stream/cam1">.

04

Log pagina

Chronologische lijst van alle events: fan, pomp, licht — allemaal met tijdstempel.

05

Emergency controls pagina

Beveiligd met wachtwoord. Knoppen voor water, licht, fan. Stuurt POST request naar Flask API.

7

Testen & Afwerken

Alles controleren en stabiel maken

Test
01

Alles testen van buiten netwerk

Via mobiel data (niet thuis wifi) controleren of website, streams en commando's werken.

02

Scripts automatisch starten bij reboot

Control script, Flask API en camera server instellen als systemd services.

03

Stabiliteitstest

Pi 24–48 uur laten draaien en controleren of alles stabiel blijft.

04

Experiment starten

Aardbeiplanten plaatsen, timelapse starten, en het experiment officieel beginnen.
